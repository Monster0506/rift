Rift — Visual Mode, Converged Candidate

Base accumulation is Model A (v / Esc to bank disjoint regions). Region navigation is split into three distinct, non-overlapping keys: n/N cycle the cursor between regions you already have, m/M bank the next/previous occurrence of the last region's text (opt-in, not default), and gv opens the region set as a real, navigable split window. o/O swap which end of the active region the cursor is on, for adjusting the other side. Regions merge only on true overlap — touching regions stay independent (see the demo-desc below for why that matters).

Try it

v + motion selects, Esc commits the region and returns to Normal (regions stay highlighted, set persists). v again starts a new disjoint region — unless the cursor is already sitting inside a banked region, in which case v instead resumes that region (pops it back out, restores its original anchor/cursor direction) so you can extend or shrink it; Esc re-commits it. Two regions only merge into one if they truly overlap (share a character) — merely touching regions (like the two "foo"s inside "foofoo" below) stay independent, so a batch Change replaces each one on its own. d/y act on every banked region as one operation. Selections span multiple lines now — j/k while in visual extends across rows, and a multi-line selection banks as a single atomic region (one entry in the regions list, however many lines it touches).
